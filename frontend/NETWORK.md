Инструкция по доступу к проекту с другого устройства

1) Запуск backend
- Откройте терминал в папке `backend`
- Установите зависимости: `npm install`
- Запустите в режиме разработки: `npm run dev`
- По умолчанию сервер слушает порт 4000 (можно изменить через переменную `PORT`)

2) Запуск frontend
- Откройте терминал в папке `frontend`
- Установите зависимости: `npm install`
- Запустите: `npm run dev`
- Vite в этой конфигурации привязан к сети (слушает на всех интерфейсах). Откройте в другом устройстве: `http://<IP_машины>:5173`

3) Как узнать IP в Windows
- Откройте PowerShell и выполните `ipconfig`
- Найдите `IPv4 Address` в интерфейсе, подключённом к вашей сети (например, Wi‑Fi или Ethernet)

4) Примечания
- Backend остаётся проксируемым локально (proxy `/api` -> `http://localhost:4000`), прокси выполняется самим Vite, так что внешний клиент обращается к `http://<IP_машины>:5173` и запросы `/api` будут перенаправлены к backend.
- Если с другого устройства не открывается, проверьте брандмауэр Windows (разрешите входящие соединения на порты 5173 и 4000) и настройки сети.
- Для production-сборки используйте `npm run build` и разместите на хостинге/сервере.
